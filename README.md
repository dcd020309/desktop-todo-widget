# Desktop Todo Demo

一个无需安装依赖、附着在 Windows 桌面层的待办小组件。

当前版本：`1.5.3`（正式版）

## 启动

首次下载后，双击 `Install-Todo.vbs`，程序会在 Windows 桌面创建 `Desktop Todo` 快捷方式。该快捷方式使用与程序托盘一致的图标，并通过 `wscript.exe` 静默启动，不显示命令行窗口。

也可以直接双击 `Start-Todo.vbs` 静默启动，但 `.vbs` 文件本身的图标由 Windows Script Host 决定，无法显示程序的自定义图标。请勿直接双击 `TodoWidget.ps1`，否则 Windows 可能显示 PowerShell 窗口。

项目不再提供 BAT 启动入口，因为 Windows 打开任何 `.bat` 文件时都会先创建 `cmd.exe` 或 Windows Terminal 窗口，无法保证完全静默。

也可以在 PowerShell 中运行：

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\TodoWidget.ps1
```

## 已实现

- 添加、完成、删除待办
- 可在待办右侧三点管理窗口中修改已添加待办的文字
- 可在三点管理窗口中删除已有详细说明，并将删除状态用于多电脑手动同步
- 拖动待办经过可插入位置时，相邻卡片会平滑向上下两侧让开并显示落点空隙
- OneDrive 设备快照只写入纯文本详细说明并使用紧凑 JSON，避免 PowerShell 文件元数据造成快照异常膨胀
- 通过 DDL 复选框决定是否为待办设置截止日期
- 显示 DDL 剩余或逾期天数
- 可为新待办选择一项“前置任务”
- 前置任务未完成时，待办以灰色“等待中”状态显示且不可勾选完成
- 前置任务完成后，等待中的待办会自动恢复为普通待办
- 可在设置中选择任务卡片的操作按钮样式：
  - 单一管理按钮：一个 `⋯` 按钮，弹窗后选择操作
  - 两个独立按钮：`↳` 前置任务和 `×` 删除按钮
- 自动阻止循环依赖；删除前置任务时自动解除关联
- 同状态、同 DDL 的未完成任务可拖动调整顺序
- 拖拽时提供源卡片淡出、目标卡片缩放和排序后入场动画
- 卡片之间的空隙也是有效拖放区域；落在卡片内部时按上下半区决定插入位置
- DDL 日期框与前置任务选择框左右对齐
- 按可配置的紧急天数阈值优先排列临近 DDL 的待办
- 通过标题栏齿轮按钮打开设置
- 设置中可选择参与普通窗口层级的“窗口模式”，或持续置于所有普通窗口最底层的“壁纸模式”
- 设置中可启用当前用户的开机自启动，并通过 VBS 静默启动组件
- 设置中可手动检查 GitHub 正式版更新；确认后自动下载、备份程序、覆盖升级并静默重启
- 设置中提供 OneDrive 手动同步：自动检测默认 OneDrive，也可自行选择同步目录
- 每台电脑使用独立设备快照，按任务最后修改时间合并，支持多电脑修改、删除记录和 TXT 详细说明同步
- 设置中分别提供“上传同步信息”和“下载同步信息”：上传只更新本机设备快照，下载只读取并合并云端数据，不会自动回传
- 每次手动同步前都会在程序目录的 `backup` 中创建完整本地备份，设置页显示永久备份的份数与总占用空间
- 可在设置中修改主界面的“我的一天”名称，修改后立即生效并自动保存
- 设置中可分别备份并清除全部已完成或全部未完成待办
- 通过卡片的管理窗口创建或编辑 TXT 详细说明；已有说明的卡片会在 `⋯` 左侧显示“文”快捷按钮
- 通过标题栏下载按钮导出 Markdown 文件
- 导出文件自动保存到程序目录下的 `output` 文件夹
- 按可选时间范围导出已完成事项、未完成事项，或同时导出两者；不选择时间时会包含没有 DDL 的未完成事项
- 开始日期留空表示从最早记录开始，结束日期留空表示不限制未来时间
- 记录事项的实际完成时间；取消完成时会清除完成时间
- “全部 / 待完成 / 已完成”筛选
- 窗口模式：组件可显示在其他普通窗口上方，切换到其他窗口后也允许被覆盖
- 壁纸模式：组件始终保持在桌面之上、所有普通应用窗口之下
- 不显示任务栏图标，也不出现在 `Alt+Tab`
- 在任务栏右下角通知区域显示托盘图标，支持定位窗口、打开设置、静默重启和退出
- 主窗口右上角提供最小化按钮，可将窗口收起到系统托盘；通过托盘双击或“定位待办窗口”恢复
- 双击托盘图标会最小化待办所在显示器上的其他普通应用窗口，让待办露出；不会移动或临时置顶待办，也不会修改窗口模式或壁纸模式
- 使用紫色圆角方块与白色勾号作为托盘和程序窗口的自定义图标
- 窗口拖动和位置锁定
- 窗口宽度固定，底部右侧可上下拖动调整高度
- 导出、设置和固定按钮提供中文悬停提示
- DDL、前置任务、筛选以及导出选项会在重启后恢复
- 添加新待办后保持当前“全部 / 待完成 / 已完成”筛选，不再自动跳回“全部”
- 即使勾选前置任务后尚未选择具体任务，重启后也会保留勾选状态
- 自动保存到 `%LOCALAPPDATA%\DesktopTodoDemo\state.json`
- 每个待办的详细说明保存到程序目录下的 `detail\待办名称_短任务编号.txt`
- 批量清除的待办及其详细说明保存到程序目录下带时间戳的 `backup` 子文件夹
- 自动记住窗口位置
- 启动时检查窗口位置；运行中如因显示器变化找不到组件，可通过系统托盘的“定位待办窗口”移回主屏
- 单实例运行，避免多个重叠窗口互相覆盖标题、设置和待办顺序

## 说明

这是一个 WPF 原型，仅支持 Windows。窗口模式使用普通窗口层级；壁纸模式以桌面系统窗口作为拥有者，并持续保持在普通应用窗口下方。两种模式均保留独立的 WPF 渲染和透明圆角。关闭按钮会退出小组件，下次启动时待办仍会保留。

Markdown 导出不依赖额外运行时。文件默认写入与 `Start-Todo.vbs` 相同目录下的 `output` 文件夹。

Markdown 导出表格包含前置任务列，等待中的未完成事项会标记为“等待中”。未选择时间范围时会导出全部未完成事项；设置时间范围后，未完成事项按 DDL 筛选，没有 DDL 的事项不会纳入该时间范围。

## OneDrive 手动同步

同步数据默认写入当前 Windows 用户的 `OneDrive\DesktopTodoSync\devices`。不同用户使用各自的 OneDrive，除非主动共享同步文件夹，否则不会互相看到待办。

每台电脑只写入以设备编号命名的独立 JSON 快照，避免多台电脑同时覆盖同一个文件。同一待办发生冲突时，以最后修改时间较晚的版本为准；删除记录会作为隐藏标记永久保留，防止旧电脑把已删除待办重新带回。

OneDrive 客户端仍负责在后台传输快照文件，但桌面待办不会自动读取或应用变化。“上传同步信息”只把本机当前数据写入本机设备快照；“下载同步信息”读取所有已到达本机 OneDrive 文件夹的设备快照，并按最后修改时间与本地数据合并，但不会自动上传合并结果。需要将下载后的合并结果提供给其他电脑时，再手动点击上传。

“上传同步信息”显示“本机写入成功”时，表示设备快照已经成功写入本机 OneDrive 同步文件夹；之后是否已经传到云端由 OneDrive 客户端负责，可通过任务栏中的 OneDrive 状态图标确认。

同步前备份永久保留在 `backup\时间戳_同步前备份`。程序不会自动删除备份；可根据设置页显示的占用空间手动清理不再需要的备份文件夹。

## 软件自动更新

在“设置 → 软件更新”中点击“检查更新”，程序才会访问 GitHub 检查默认 `main` 分支的正式版，不会在后台定时轮询。检查过程在后台执行，连接超时或失败不会卡住设置窗口。发现更高版本后会先询问是否更新；确认后，独立的静默更新程序会下载最新 ZIP、校验版本和必要文件、等待旧程序完全退出、备份当前程序、覆盖升级并自动重启。

更新不会覆盖 `%LOCALAPPDATA%\DesktopTodoDemo\state.json`，也会保留程序目录中的 `detail`、`backup` 和 `output` 文件夹。更新前的程序文件保存在 `backup\时间戳_程序更新前备份`；如果下载或安装失败，更新程序会自动回滚并重新启动原程序，同时将错误写入 `%LOCALAPPDATA%\DesktopTodoDemo\update.log`。

此功能需要电脑能够访问 GitHub。只有已经上传并合并到 GitHub 默认 `main` 分支的版本才会被识别为正式更新。

版本更新内容请查看 `CHANGELOG.md`。每次更新需要同步修改 `VERSION`、程序内版本号和更新日志。
