# Desktop Todo Demo

一个无需安装依赖、附着在 Windows 桌面层的待办小组件。

当前版本：`0.20.0`

## 启动

双击 `Start-Todo.vbs`。这是推荐入口，启动过程不会出现命令行窗口。

`Start-Todo.bat` 仅作为兼容入口保留；Windows 打开任何 `.bat` 文件时都会创建 `cmd.exe`，因此无法从 BAT 文件本身彻底消除瞬间闪烁。

也可以在 PowerShell 中运行：

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\TodoWidget.ps1
```

## 已实现

- 添加、完成、删除待办
- 通过 DDL 复选框决定是否为待办设置截止日期
- 显示 DDL 剩余或逾期天数
- 可为新待办选择一项“前置任务”
- 前置任务未完成时，待办以灰色“等待中”状态显示且不可勾选完成
- 前置任务完成后，等待中的待办会自动恢复为普通待办
- 可在设置中选择任务卡片的操作按钮样式：
  - 单一管理按钮：一个 `⋯` 按钮，弹窗后选择操作
  - 两个独立按钮：`↳` 前置任务和 `×` 删除按钮
- 自动阻止循环依赖；删除前置任务时自动解除关联
- 同状态、同 DDL 的未完成任务可拖动调整顺序
- 拖拽时提供源卡片淡出、目标卡片缩放和排序后入场动画
- 卡片之间的空隙也是有效拖放区域；落在卡片内部时按上下半区决定插入位置
- DDL 日期框与前置任务选择框左右对齐
- 按可配置的紧急天数阈值优先排列临近 DDL 的待办
- 通过标题栏齿轮按钮打开设置
- 设置中可选择“窗口模式”或持续置于所有普通窗口最底层的“壁纸模式”
- 设置中可启用当前用户的开机自启动，并通过 VBS 静默启动组件
- 可在设置中修改主界面的“我的一天”名称，修改后立即生效并自动保存
- 设置中可分别备份并清除全部已完成或全部未完成待办
- 通过卡片的管理窗口创建或编辑 TXT 详细说明；已有说明的卡片会在 `⋯` 左侧显示“文”快捷按钮
- 通过标题栏下载按钮导出 Markdown 文件
- 导出文件自动保存到程序目录下的 `output` 文件夹
- 按可选时间范围导出已完成事项、未来计划，或同时导出两者
- 开始日期留空表示从最早记录开始，结束日期留空表示不限制未来时间
- 记录事项的实际完成时间；取消完成时会清除完成时间
- “全部 / 待完成 / 已完成”筛选
- 桌面插件模式：普通应用会覆盖组件
- 始终保持在桌面之上、普通应用之下
- 不显示任务栏图标，也不出现在 `Alt+Tab`
- 在任务栏右下角通知区域显示托盘图标，支持定位窗口、打开设置和退出
- 使用紫色圆角方块与白色勾号作为托盘和程序窗口的自定义图标
- 窗口拖动和位置锁定
- 窗口宽度固定，底部右侧可上下拖动调整高度
- 导出、设置和固定按钮提供中文悬停提示
- DDL、前置任务、筛选以及导出选项会在重启后恢复
- 即使勾选前置任务后尚未选择具体任务，重启后也会保留勾选状态
- 自动保存到 `%LOCALAPPDATA%\DesktopTodoDemo\state.json`
- 每个待办的详细说明保存到程序目录下的 `detail\待办名称_短任务编号.txt`
- 批量清除的待办及其详细说明保存到程序目录下带时间戳的 `backup` 子文件夹
- 自动记住窗口位置
- 持续检查显示器布局；副屏断开或切换复制模式后，自动将屏幕外的组件移回主屏
- 单实例运行，避免多个重叠窗口互相覆盖标题、设置和待办顺序

## 说明

这是一个 WPF 原型，仅支持 Windows。组件以桌面系统窗口作为拥有者，但保留独立的 WPF 渲染，因此普通应用会覆盖它且透明圆角仍能正常显示。关闭按钮会退出小组件，下次启动时待办仍会保留。

Markdown 导出不依赖额外运行时。文件默认写入与 `Start-Todo.vbs` 相同目录下的 `output` 文件夹。

Markdown 导出表格包含前置任务列，等待中的未来计划会标记为“等待中”。

版本更新内容请查看 `CHANGELOG.md`。每次更新需要同步修改 `VERSION`、程序内版本号和更新日志。
