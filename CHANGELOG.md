# 更新日志

本项目使用语义化版本号。后续每次功能或修复更新，都需要同时修改 `VERSION`、程序内版本号和本文件。

## 0.18.0 - 2026-07-15

- 新增显示器布局监测，每 2 秒检查组件是否仍位于当前有效屏幕范围内。
- 修复组件停留在副屏时，通过 `Win + P` 切换为复制模式或断开副屏后无法找到窗口的问题。
- 当组件与所有有效屏幕均无足够可见区域时，自动将其移动到主屏工作区中央并保存新位置。
- 窗口层级模式、位置锁定状态和自动移回主屏功能可同时正常工作。

## 0.17.0 - 2026-07-15

- 设置页面新增“数据管理”区域，可分别备份并清除全部已完成待办或全部未完成待办。
- 每次清除前显示数量并要求确认；备份失败时不会从主列表移除任何待办。
- 备份保存到 `Start-Todo.vbs` 同目录的 `backup\时间戳_分类` 文件夹。
- 每份备份包含完整的 `tasks.json`，有关联详细说明时同时复制到备份内的 `detail` 文件夹。
- 备份成功后才从当前数据和 `detail` 文件夹移除对应内容，并自动解除剩余待办对已清除任务的前置依赖。

## 0.16.0 - 2026-07-15

- 将待办卡片的详细说明按钮移动到三点管理按钮左侧。
- 详细说明文件改用“待办名称 + 8 位短任务编号”的可读文件名，例如 `准备周报_a1b2c3d4.txt`。
- 自动替换 Windows 文件名中的非法字符，并限制名称长度，避免创建文件失败。
- 打开旧版说明时自动将纯任务 ID 文件迁移为新名称，保留原有内容。

## 0.15.0 - 2026-07-15

- 重写同时导出两类事项时的说明文字，明确已完成事项按完成日期筛选、未来计划按 DDL 日期筛选。
- 待办管理窗口新增“编辑详细说明”按钮。
- 首次编辑时在 `Start-Todo.vbs` 所在目录创建 `detail` 文件夹，并以任务 ID 建立独立 TXT 文件后用记事本打开。
- 已有详细说明的待办卡片右侧显示“文”按钮，可直接再次打开对应 TXT 文件。
- 详细说明文件与任务数据分开保存，编辑内容由记事本直接写入。

## 0.14.0 - 2026-07-15

- Markdown 导出的“已完成事项”和“未来计划”改为两个独立复选框，支持同时导出。
- 同时导出时，时间范围分别应用于已完成事项的完成时间和未来计划的 DDL。
- 新增单实例保护，阻止多个桌面待办窗口重叠运行。
- 修复多个实例周期性争夺窗口层级，造成旧名称与新名称来回显示的问题。
- 修复多个实例交替显示不同任务顺序，造成拖拽排序后卡片来回跳转的问题。
- 导出类型选择会继续保存，并支持恢复“两者同时导出”。

## 0.13.0 - 2026-07-15

- 设置页面新增名称输入框，可修改主界面默认的“我的一天”标题。
- 名称修改后立即显示，并保存到状态文件供下次启动恢复。
- 名称不能为空，最长 30 个字符；较长名称会在主界面自动显示省略号，避免覆盖右侧按钮。

## 0.12.0 - 2026-07-15

- 设置页面新增“开机自启动”选项。
- 启用后为当前 Windows 用户写入登录启动项，并通过 `Start-Todo.vbs` 静默启动组件；关闭后自动移除启动项。
- 自启动选择会保存到状态文件，程序路径变化后手动启动一次可自动刷新启动项路径。
- 修改启动项不需要管理员权限；写入失败时会在设置窗口显示原因。

## 0.11.1 - 2026-07-15

- 将窗口层级的两个设置名称简化为“窗口模式”和“壁纸模式”。
- 仅调整显示名称，原有窗口层级行为和状态保存逻辑保持不变。

## 0.11.0 - 2026-07-15

- 设置页面新增“窗口层级”，可选择“桌面插件模式”或“始终置于所有普通窗口的最底层”。
- 桌面插件模式保持原有行为；始终最底层模式会在程序运行期间持续校正窗口层级。
- 窗口层级选择会保存到状态文件，并在下次启动时自动恢复。

## 0.10.2 - 2026-07-15

- 修复卡片间距视觉上可见、但没有参与 WPF 鼠标命中测试的问题。
- 为待办列表设置透明背景，使卡片之间的空白区域成为真实的可拖放区域。
- 保留列表级目标计算和卡片上下半区插入逻辑。

## 0.10.1 - 2026-07-15

- 将拖放接收逻辑从单张卡片提升到整个待办列表，减少拖动过程中出现禁止光标的情况。
- 卡片之间的空白间距现在也是有效放置区域。
- 在两张同级卡片之间松开鼠标时，将拖动项准确插入两者之间。
- 在卡片内部松开鼠标时，根据指针位于卡片上半区或下半区，分别插入卡片之前或之后。
- 在不同状态或 DDL 区域移动时仍保持原有同级排序限制，但列表内持续显示可移动光标并解析到最近的有效同级位置。
- 目标卡片缩放提示改由列表级拖放逻辑统一管理，避免卡片与间隙之间切换时动画闪烁。

## 0.10.0 - 2026-07-15

- 将右上角设置按钮的悬停提示改为“打开待办设置”。
- 设置窗口中的“待办设置”和“卡片操作方式”统一为 16 像素、半粗体和相同文字颜色。
- 设置选项文字移除“（v0.8 样式）”和“（v0.7 样式）”后缀。
- 拖拽开始时，源卡片会平滑淡出并轻微缩小。
- 拖动经过有效目标卡片时，目标卡片会轻微放大提示可放置位置。
- 离开或放置后目标卡片恢复原尺寸。
- 排序刷新后，所有卡片使用淡入和向上移动的入场动画。

## 0.9.0 - 2026-07-15

- 修复“前置任务”已勾选但尚未选择具体任务时，重启后被自动取消勾选的问题。
- 前置任务列表为空或未选中时不再强制修改复选框状态；真正添加任务时再提示选择。
- 设置窗口新增“卡片操作方式”选项。
- 支持 `单一管理按钮（v0.8 样式）`：卡片显示一个 `⋯` 按钮，点击后在管理窗口选择操作。
- 支持 `前置任务和删除两个按钮（v0.7 样式）`：卡片直接显示 `↳` 和 `×` 两个按钮。
- 卡片操作方式保存到 `state.json`，修改后立即刷新，并在重启后恢复。
- 抽取统一的任务删除逻辑，确保两种按钮样式都会正确解除相关依赖。

## 0.8.0 - 2026-07-15

- 将 DDL 和前置任务两行的标签宽度统一为 72 像素，日期框与选择框宽度统一为 156 像素，实现左右对齐。
- 新增 `sortOrder` 顺序字段，并自动迁移旧任务。
- 支持拖动同状态、同 DDL 的未完成任务调整顺序；不同 DDL、等待状态或已完成任务之间不会误排序。
- 拖动后的顺序会写入本地状态文件并在重启后恢复。
- 将卡片右侧的前置任务和删除按钮合并为一个 `⋯` 管理按钮。
- 管理弹窗中可选择“修改或取消前置任务”或“删除待办”，删除前会二次确认。
- 新增界面状态持久化，保存 DDL 开关和日期、前置任务开关和选项、当前筛选按钮、导出类型及导出日期范围。
- 添加待办后不再重置 DDL 和前置任务选项，方便连续创建同类事项。
- 重新启动后恢复上述界面状态；无效或已删除的前置任务选项会安全取消。

## 0.7.0 - 2026-07-15

- 新增“前置任务”功能，可在创建待办时选择必须先完成的现有任务。
- 前置任务未完成时，依赖项进入“等待中”状态：卡片灰显、复选框禁用，但不使用已完成事项的删除线。
- 卡片会显示正在等待的前置任务；前置任务完成后依赖项会自动恢复为普通待办。
- 每张任务卡片新增 `↳` 按钮，可在创建后修改或取消前置任务。
- 编辑器会排除可能形成循环依赖的候选项，并在保存时再次校验。
- 删除一项前置任务时，所有依赖该任务的待办会自动解除关联，避免永久阻塞。
- 等待中的任务排在普通未完成任务之后、已完成任务之前。
- Markdown 导出增加“前置任务”列，并将未解除的依赖项标记为“等待中”。
- 新增区域高度调整为适配 DDL 和前置任务两行选项，默认窗口高度更新为 610 像素。

## 0.6.0 - 2026-07-15

- 导出格式由 Excel `.xlsx` 改为 Markdown `.md`。
- 导出文件默认写入 `Start-Todo.vbs` 同目录下的 `output` 文件夹，不再弹出保存位置窗口。
- 导出开始日期和结束日期默认均为空。
- 开始日期为空时从最早记录开始，结束日期为空时不限制未来时间，也支持仅设置一侧日期。
- 新增 DDL 启用复选框，默认不勾选；未勾选时新待办不包含 DDL。
- 勾选 DDL 后启用日期选择器，取消勾选会清空日期。
- 主窗口宽度固定为 380 像素，支持从右下角纵向调整高度，并保存调整后的高度。
- 为导出、设置和固定按钮增加更明确、响应更快的中文悬停提示。
- 移除旧的 Excel 生成器和 Node.js 运行时依赖。

## 0.5.0 - 2026-07-15

- 修复点击待办复选框后完成状态未写回的问题。
- 复选框和删除按钮改为基于 WPF 事件 `sender` 识别任务，移除不稳定的 `$this` 和闭包依赖。
- 新增 `completedAt` 完成时间字段；勾选时记录，取消勾选时清除。
- 旧版已完成事项会以创建时间作为兼容的完成时间。
- 新增 Excel 导出窗口，可选择开始日期、结束日期和导出内容。
- 支持按完成时间导出所选范围内的全部已完成事项。
- 支持按 DDL 导出所选范围内尚未完成的未来计划。
- 导出文件为标准 `.xlsx`，包含标题、筛选表格、日期格式和冻结表头。
- Excel 生成过程在后台运行，不显示额外命令行窗口。
