# 更新日志

本项目使用语义化版本号。后续每次功能或修复更新，都需要同时修改 `VERSION`、程序内版本号和本文件。

## 1.5.0 - 2026-07-16

- 设置页面新增“软件更新”区域，可手动检查 GitHub 默认 `main` 分支上的正式版；不进行后台定时检查。
- 发现新版本后可直接确认更新，无需手动下载 ZIP 或解压；独立更新程序会在主程序退出后静默下载、安装并重启。
- 更新包会校验目标版本和必要程序文件，避免下载不完整或版本不一致的内容。
- 更新前自动将现有程序文件备份到 `backup\时间戳_程序更新前备份`，安装失败时自动回滚。
- 升级过程中保留本地待办状态，以及程序目录中的 `detail`、`backup` 和 `output` 用户数据文件夹。
- 新增 `%LOCALAPPDATA%\DesktopTodoDemo\update.log` 更新日志，便于定位网络、下载或安装失败原因。

## 1.4.0 - 2026-07-16

- 将设置页面原来的“立即同步”拆分为“上传同步信息”和“下载同步信息”两个独立按钮。
- 上传只备份并写入当前电脑对应的设备快照，不读取或应用其他电脑的改动。
- 下载会先备份本机，再读取所有有效云端设备快照并与本地数据合并；下载结果不会自动回传 OneDrive。
- 两种方向继续使用任务最后修改时间解决冲突，并同步删除记录和 TXT 详细说明。
- 同步状态文字会明确显示当前操作方向、上传的待办与删除记录数量，或下载合并的云端设备快照数量。
- 上传前备份和下载前备份使用不同的文件夹名称及备份原因，方便手动识别和恢复。

## 1.3.3 - 2026-07-16

- 新增 `Install-Todo.vbs`：一键在当前用户桌面创建 `Desktop Todo` 快捷方式，统一使用 `assets/todo-icon.ico`。

- 删除 `Start-Todo.bat`，从分发包中移除无法保证完全静默的入口；README 增加首次安装和正确启动说明。

## 1.3.2 - 2026-07-16

- 修复 OneDrive 设备快照可能从几 KB 异常膨胀到约 100 MB 的问题。
- 根因是 Windows PowerShell 为 `Get-Content` 返回值附加了 `PSPath`、`PSDrive`、`PSProvider` 等文件系统元数据，序列化时被递归写入 JSON。
- 详细说明改用 .NET 纯文本读取，确保 `detailContent` 始终是普通字符串；同步快照同时改用紧凑 JSON。
- 兼容旧版异常快照：检测到对象形式的 `detailContent` 时提取其中的 `value` 正文，避免说明内容变成对象描述文字。
- 现有异常快照可在备份后安全重写，待办、删除记录及详细说明正文保持不变。
